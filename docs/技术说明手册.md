## 1. 项目概述

### 1.1 定位与目标

Taiji Agent 2.0 是一个基于**太极哲学**的 AI Agent 框架，融合了以下开源框架的核心能力：

| 来源框架 | 贡献能力 |
|---------|---------|
| **Hermes Agent** | 54+工具、记忆系统、Skills Hub、自我学习闭环 |
| **cgast/harness** | Agent Loop 核心引擎 (~350行)、Soul 人格系统 |
| **OpenTaiji** | Taiji Verify 防幻觉系统、太极思维框架 |
| **Dify/MCP** | MCP 双向协议集成、人机协作 (HITL) |
| **LangGraph** | 状态工作流引擎 |
| **OpenAI Agents SDK** | Guardrails 安全护栏、Agent Handoffs |

### 1.2 设计哲学

```
┌─────────────────────────────────────────────────────────────────┐
│                     太极哲学三原则                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   道生一 ──────────────────────────────────────────────────────  │
│    │                                                           │
│   一生二 ─ 阳（确定性） ◄────────────► 阴（创造性）               │
│    │         Taiji Verify验证                  LLM生成                  │

└─────────────────────────────────────────────────────────────────┘
```

**三大设计原则**：

| 原则 | 含义 | 工程体现 |
|------|------|---------|
| **分布式** | 无上帝节点，每个 Agent 独立主体 | 事件驱动、消息传递、无共享状态 |
| **可进化** | 系统不是"写完"的，是"长出来"的 | 自我学习闭环、记忆动态增长 |
| **自组织** | 无中央调度，智能体自决策 | Skills Hub 自创建、任务自规划 |

## 3. 核心模块详解

### 3.1 Agent Loop 引擎

**文件**: `src/taiji_agent/agent/engine.py`

Agent Loop 是框架的核心执行引擎，基于 cgast/harness 设计，约 350 行代码实现。

#### 核心类

```python
class TaijiAgent:
    """太极 Agent 核心"""
    
    def __init__(
        self,
        config: AgentConfig | None = None,
        provider: LLMProvider | None = None,
    ):
        self.config = config or AgentConfig()
        self.soul_loader = SoulLoader()
        self.Taiji Verify = Taiji VerifyVerifier()
        self.hallucination_detector = HallucinationDetector()
        self.memory = SessionMemory()
        self.tools = ToolRegistry()
        self.event_bus = EventBus()
```

#### 执行流程

```
run(task) → Agent Loop
    │
    ├─► _build_system_prompt()     # 组装Soul + Taiji Verify指南
    │
    ├─► Loop (max_iterations)
    │       │
    │       ├─► _assemble_prompt()     # 组装消息历史
    │       │
    │       ├─► provider.chat()        # LLM调用
    │       │
    │       ├─► _verify_and_annotate() # Taiji Verify验证
    │       │
    │       ├─► tools.execute()        # 工具执行（如有tool_calls）
    │       │
    │       └─► memory.save_session()   # 保存记忆
    │
    └─► TaskResult
```

#### 使用示例

```python
from taiji_agent import TaijiAgent, AgentConfig
import asyncio

async def main():
    config = AgentConfig(
        provider="anthropic",
        model="claude-sonnet-4-20250514",
        Taiji Verify_enabled=True,
        max_iterations=25,
    )
    
    agent = TaijiAgent(config=config)
    result = await agent.run("帮我分析代码性能")
    
    print(f"状态: {result.status}")
    print(f"内容: {result.content}")
    print(f"迭代: {result.iterations}")

asyncio.run(main())
```

### 3.2 Taiji Verify 防幻觉系统

**文件**: `src/taiji_agent/Taiji Verify/verifier.py`

Taiji Verify (Witness & Fact Grounded Verifier) 是项目的核心差异化能力，提供符号层防幻觉验证。

#### 组件架构

```
Taiji VerifyVerifier ─────────────────────────────┐
     │                                     │
     ├─► SelfConsistencyChecker           │  # 多路径自检
     │        │                           │
     │        └─► 多次采样 + 投票          │
     │                                     │
     ├─► HallucinationDetector            │  # 幻觉检测
     │        │                           │
     │        └─► 综合评分 = Taiji Verify(40%)    │
     │                   + 自检(30%)      │
     │                   + 溯源(30%)      │
     │                                     │
     └─► SourceTracer                    │  # 知识溯源
              │
              └─► 结论 → 原始来源映射
```

#### 使用示例

```python
from taiji_agent import Taiji VerifyVerifier, HallucinationDetector

# 符号层验证
verifier = Taiji VerifyVerifier()
passed = verifier.verify("LLM的输出内容")

# 幻觉风险检测
detector = HallucinationDetector()
risk = detector.detect("可能的回复内容")
print(f"幻觉风险: {risk:.1%}")

# 风险阈值控制
if risk > 0.3:
    print("需要重写或添加警告")
```

### 3.3 Soul 人格系统

**文件**: `src/taiji_agent/souls/loader.py`

Soul 系统允许通过 YAML 文件定义 Agent 的人格特征。

#### YAML 格式

```yaml
# souls/default.yaml
id: default
name: "太极助手"

layers:
  # 边界层 - 绝对不可逾越
  boundaries:
    - "不产生有害内容"
    - "永远坦诚承认不确定性"
    - "不冒充人类"
  
  # 伦理层 - 道德准则
  ethics:
    - "追求阴阳平衡"
    - "尊重用户意图"
    - "保护隐私"
  
  # 性格层 - 行为特征
  character:
    traits:
      - "深思熟虑"
      - "善于分析"
      - "简洁表达"
    tone:
      - "专业"
      - "友好"
      - "中立"
```

#### 使用示例

```python
from taiji_agent.souls.loader import SoulLoader, inject_soul

loader = SoulLoader()
soul = loader.load("default")

# 获取人格提示
prompt = inject_soul(soul)
print(prompt)
```

### 3.6 MCP 协议集成

**文件**: `src/taiji_agent/mcp/`

将 Agent 发布为 MCP Server，供外部 MCP Client 调用：

```python
from taiji_agent.mcp.server import MCPServerAdapter, MCPServerConfig

config = MCPServerConfig(
    host="0.0.0.0",
    port=8080,
    server_name="taiji-agent",
)

adapter = MCPServerAdapter(agent, config)
adapter.register_agent_as_tool()
url = await adapter.start()
```

#### MCP Client

连接外部 MCP Server 作为工具：

```python
from taiji_agent.mcp.client import MCPClientAdapter, MCPConnectionConfig

config = MCPConnectionConfig(
    url="http://mcp-server:8080/message",
    name="external_server",
)

client = MCPClientAdapter(config)
await client.connect()
tools = client.get_tools()
```

### 3.9 多智能体协同

| `BROADCAST` | 广播 | 通知、公告 |
| `DEBATE` | 辩论讨论 | 方案评审 |
| `CONSENSUS` | 共识投票 | 团队决策 |

```python
from taiji_agent.multiagent.coordinator import (
    MultiAgentCoordinator,
    AgentRole,
    CoordinationMode,
)

coord = MultiAgentCoordinator(
    max_concurrent=3,
    max_depth=2,

result = await coord.execute_hierarchical(task)
```

### 4.3 工具清单 (20个)

| `approval_submit` | 提交审批 | `request_id: str` |
| `approval_approve` | 审批通过 | `request_id, approver_id, comment` |
| `approval_reject` | 审批拒绝 | `request_id, approver_id, comment` |
| `approval_status` | 查询审批状态 | `request_id: str` |

#### 审计日志工具 (3个)

| 工具名 | 说明 | 参数 |
|--------|------|------|
| `audit_log` | 记录审计日志 | `action, user_id, resource, details` |
| `audit_query` | 查询审计日志 | `user_id, action, limit` |
| `audit_verify` | 验证审计链 | (无参数) |

#### 政务数据工具 (6个)

| 工具名 | 说明 | 参数 |
|--------|------|------|
| `mask_id_number` | 身份证号脱敏 | `id_number: str` |
| `mask_phone` | 手机号脱敏 | `phone: str` |
| `mask_bank_card` | 银行卡号脱敏 | `bank_card: str` |
| `validate_id_number` | 身份证号验证 | `id_number: str` |
| `validate_credit_code` | 信用代码验证 | `credit_code: str` |
| `calculate_workday` | 工作日计算 | `start_date, days` |

### 4.4 使用方式

#### 方式一：直接使用

```python
from taiji_agent.govmcp.server import GovMCPServer

server = GovMCPServer()

# 调用工具

```

```python
from taiji_agent.gov_enhanced_engine import GovEnhancedHermesEngine

engine = GovEnhancedHermesEngine()
await engine.enable_govmcp()

# 加密数据
encrypted = await engine.encrypt_data("敏感政务数据")

    title="数据访问申请",
    description="申请访问加密数据",
    requester="user001",
    department="政务中心",

# 批准审批
await engine.approve_approval(
    request_id=approval["request_id"],
    approver_id="manager001",
)
```
